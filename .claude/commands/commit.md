Review the current git diff and create a commit with an appropriate message.

Steps:
1. Check changes with `git status` and `git diff`
2. Analyze changes and create a commit message in Conventional Commits format
   - feat: New feature
   - fix: Bug fix
   - refactor: Refactoring
   - docs: Documentation
   - test: Tests
   - chore: Other changes
3. Stage all changes: `git add -A`
4. Create the commit

Commit messages must be written in English. Keep the message concise and descriptive.
